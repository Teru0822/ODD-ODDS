# ルール遵守検証

## 検証の目的
直近の作業内容や指定されたファイルが、プロジェクトのコーディング規約、Unityルール、Gitルールなどの各種ガイドライン（`Docs/01_Rules_and_Policies/...`）に適合しているかチェックします。

## 【検証手順】
1. 検証対象となるファイル、または直近の修正内容を確認します。
2. `Docs/01_Rules_and_Policies/02_Development_Guidelines/CodingRules.md` や `UnityRules.md` などのルールと照らし合わせ、以下の観点で評価・レポートします。
   - ルール違反・逸脱はあるか（命名規則、ファイル分割、コメント等）
   - Git/タスク運用ルールに則っているか
3. 結果をユーザーへ報告し、必要であれば修正案を提示してください。
