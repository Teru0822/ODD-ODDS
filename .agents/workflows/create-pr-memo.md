# PR メモ自動生成

## 目的
開発者がブランチでの作業を終えてプルリクエスト（PR）を作成する前に、変更内容の差分（`git diff` または作業ログ）を分析し、プロジェクトの「PR / レビュー方針」に沿った説明文の雛形を自動作成します。

## 【実行手順】
1. 現在の未コミット差分、または直近のコミット履歴（`git log`や`git diff`など）を確認します。またはユーザーから与えられた作業ログを読み込みます。
2. `Docs/01_Rules_and_Policies/03_Workflows/GitWorkflow.md`内にある「PR / レビュー方針」に従い、以下の項目を埋めたマークダウン形式のPR説明文を出力してください。
   - **目的**: 何を達成するための変更か
   - **変更内容**: 実装・修正した具体的な内容
   - **対象ファイル**: 変更が加わった主なファイル
   - **確認内容**: 動作確認・テストしたこと
   - **懸念点 / 未確認事項**: まだ確認できていないこと、注意点など
3. 生成されたマークダウンテキストをユーザーにそのままコピペできる形で提示してください。
