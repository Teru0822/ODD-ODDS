# TODO・技術的負債マッピング

## 目的
プロジェクト内（特に `*.cs` などのソースコード）に残された `TODO:`, `FIXME:`, `HACK:` などのコメントを一斉検索し、現在のやり残しタスクや技術的負債を把握しやすくリスト化します。

## 【実行手順】
1. `grep_search` 等を用いて、プロジェクト内のスクリプトファイルを中心に「TODO」「FIXME」「HACK」の大文字小文字を問わないキーワード検索を実行します。
2. 検出された行とそのファイルパス、可能であれば簡潔な周辺の文脈を取得します。
3. 取得した一覧をMarkdownのチェックボックスリスト（`- [ ]`）形式に成形してユーザーに報告します。
4. もし何も見つからなければ「現在のところ TODO や FIXME は見つかりませんでした。健全な状態です！」と報告してください。
