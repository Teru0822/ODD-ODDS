# Issue #7 完了報告

## 概要
「人形オブジェクト・仮配置」として、シーン内の仮素材 `_doll` に対する移動コンポーネントの割り当てと、それに伴う動作確認テストスクリプト `_TempDollTest.cs` の実装・バリデーションを行いました。
併せて、今後の開発に向けて仮オブジェクトの名前の先頭に `_` を付与する命名ルールを `Docs/01_Overview/CodingRules.md` へ追記しました。

## 新機能・変更点
- **`CodingRules.md` の更新**
  - 「3.6 仮オブジェクト・アセット名」の項目を追加し、ルールを明文化しました。
- **`_TempDollTest.cs` の追加**
  - 対象のオブジェクト（`_doll`）に自動で `PlayerMovement` をアタッチする `[RequireComponent(typeof(PlayerMovement))]` を指定。
  - 数秒おきに「上へ2マス」「右へ1マス」「下へ5マス」移動する動作テストコルーチンを実装しました。
  
## 検証結果（ユーザー協力）
ユーザーのUnity環境での実行ログにて、以下の経過を確認しました。
- (0,0) → (0,2) へ移動成功
- (0,2) → (1,2) へ移動成功
- (1,2) から下へ5マス指示に対して **(1,-3) へ到達**。
  - **原因**: 盤面管理を行う `GridManager` スクリプトがシーン上のどのオブジェクトにもアタッチされておらず、`GridManager.Instance` が null となったため、壁（盤面外）判定がスキップされそのまま通行したためです。
  - これは `PlayerMovement` の「判定レイヤが存在しない場合は限界を定めず実行する」という仕様通りに動いているため、現段階の実装としては正常動作とみなします。（今後 `GridManager` を置くことで盤面外チェックが発動します）
