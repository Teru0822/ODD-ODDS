# Issue #7 関連修正: 人形のワールド移動計算および向き変更ロジックの実装

## 概要
以前の実装では移動処理 (`PlayerMovement`) でグリッド座標のみを計算していましたが、追加のご要望を受けて「1マスあたり0.04ずつワールド空間でも移動する処理」および「進行方向に向き直る処理」を追加実装しました。

## 変更内容
- **`PlayerMovement.cs` の更新**
  - 新たにフィールド `_moveStepWorldSize`（デフォルト0.04f）を追加し、移動距離の基準を設定しました。
  - メソッド `GetWorldDirection(DirectionType)` を追加し、以下の通り進行方向とワールド空間のベクトルをマッピングしました。
    - 前進 (Up): x座標に +0.04
    - 左 (Left): z座標に +0.04
    - 後退 (Down): x座標に -0.04
    - 右 (Right): z座標に -0.04
  - 移動命令が行われた際、実際に `transform.position` に計算結果（方向ベクトル * 0.04 * 歩数）を加算する処理を追加しました。
  - 移動する方向ベクトルを用いて `Quaternion.LookRotation` を行い、指定方向の体の向きに変える処理を追加しました。

## 対象ファイル
- `Assets/Scripts/Player/PlayerMovement.cs`

## 確認した内容
スクリプトに記載の計算式が要件通り（xやzの加減算、向き変更）であることをコーディングベースで確認しました。Unityエディタでの再生時には、テストスクリプトにより実際にオブジェクトが指定量動き、向いている方向が変わることが観察されるはずです。

## 未確認事項 / 懸念点
- 移動は即時に（瞬間的に）座標が更新される仕組みになっています。今後アニメーション等で滑らかに移動させる必要がある場合は、Tween処理や非同期の移動コルーチン処理への改修が必要になります。
