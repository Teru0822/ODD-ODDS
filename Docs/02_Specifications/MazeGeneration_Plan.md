# 巨大迷宮「4大エリア×9分割ランダム生成」設計・構築プラン

## 1. コンセプトと要件
- **全体サイズ**: 1080×540マス（割り切れる綺麗な数字として仮決定）。
- **エリア大分割（チーム分担）**: 全体を2×2の「4つの大エリア」に分割（1エリアあたり 540×270マス）。これを4人のメンバーで1エリアずつ手分けして担当します。
- **小ブロック分割（ランダム要素）**: 担当する1つの大エリアの中を、さらに3×3の「9つの小ブロック」に分割（1ブロックあたり 180×90マス）。
- **ランダム生成**: 各担当者は自分のエリア用に「9個以上」の小ブロックを作成。プレイの度にこの小ブロックをランダムに配置・入れ替えることで、毎回異なる迷宮を生み出します。
- **作画ツール**: アセット「Qubic - Modular Builder」を使用し、直感的にマップを構築。
- **設計図の生成**: 通路（直線と分岐が面白いバランスで混ざった十字路等）の設計図はAI側（スクリプト）が生成・提供します。
- **ギミック配置**: イベントマスやゴールマスは、作成後にUnityのシーンビュー・インスペクター上から「マウスクリックで直感的に増減・変更」できるようにします。

---

## 2. 【絶対ルール】小ブロックの接続点
小ブロックをどうランダムに並べ替えても道が繋がるよう、**全ブロックの出入り口の座標を完全に統一**します。
1つの小ブロックのサイズを「横180マス × 縦90マス」とした場合：
- **上の出口**: X=90 (中央)
- **下の出口**: X=90 (中央)
- **右の出口**: Y=45 (中央)
- **左の出口**: Y=45 (中央)
必ずこの座標を「通路」にするルールを徹底します。これにより、誰が作ったブロックが隣に来ても道が綺麗に繋がります。

## 3. 「設計図（迷路の道）」の生成方式：画像確認 ＋ ギズモ(ホログラム)下書き
手作業での「迷路の構造を考えながらの配置」は非常に負担が大きいため、以下の**「2段階ステップ」**で進めます。

### ステップ1：AIによる画像（見取り図）の出力と確認
まず、私（AI）がアルゴリズムを用いて迷路の配列データを生成し、**「2Dの画像データ（PNG等）」として出力**します。
これを見てチームで「直線の割合がちょうどいいか」「分岐の面白さはあるか」等のフィードバックを行い、微調整を繰り返します。

### ステップ2：合格した迷路を「Gizmo（ホログラム）」としてUnityに超軽量表示
画像案にOKが出たら、そのまま目で見てなぞるのではなく、**私が作成する下書きスクリプトがその配列データを読み込み、Unityのシーン上に『半透明のマスの色分けホログラム（Gizmo）』として道や壁を空中に映し出します。**

### 【実際の手順・Gizmo表示の圧倒的メリット】
- **ホログラム（Gizmo）とは**: 実体を持つCube（GameObject）を数万個も生成するとUnityが激しく重くなりますが、Gizmoは単なる「エディタ画面上の線描画」なので**パソコンへの負荷がほぼゼロ**です。Hierarchyも汚れずサクサク動きます。
- **ユーザーの作業**:
  1. ホログラムの下書き（例：赤いマス＝壁、青いマス＝道）が見えているSceneを開きます。
  2. Qubicアセットのグリッド機能を起動し、好きな「壁」や「床」のプレハブを選びます。
  3. ホログラムの色の通りになぞるように、Qubicで上からペイント（配置）していきます。
  4. ホログラムは透けて見えるため置き間違いが起きにくく、迷路の構造を考える必要なく装飾とデザインに集中できます！

## 4. イベント・ゴールマスの「クリック編集」システム
作成したマップに対して、後から誰でも簡単にイベントマス（宝箱など）やゴールを追加できるように、専用の**「MapEditorGizmo（エディタ拡張）」**を作成します。

### 仕様イメージ
1. Unityのシーンビュー上に、マップのグリッド（網目）が表示されます。
2. 画面上のボタン（UIパネル）で「イベントマス」「トラップ」「ゴール」を選びます。
3. **シーンビューのマスを直接クリック（ペイント感覚）する**と、そこにイベント情報の見えないマーカーデータ（または専用Prefab）が設置・削除されます。
4. プログラムは実行時、このマーカーを参照して実際のトラップやイベントを発生させます。
