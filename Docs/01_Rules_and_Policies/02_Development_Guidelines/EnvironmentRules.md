# 開発環境共通ルール (Environment Rules)

本プロジェクトは Windows / Mac の両環境で開発を行うため、環境差異による問題を避けるために以下を守ること。

## 1. 文字コード
- テキストファイルの文字コードは原則として UTF-8 を使用する。
- Shift-JIS など、環境依存で文字化けしやすい文字コードは使用しない。
- README、Markdown、ソースコード、設定ファイル、JSON、YAML、CSV、TSV、テキストメモなどは UTF-8 で保存する。
- 文字コードの変更だけを目的とした不要な差分は作らない。

## 2. 改行コード
- 改行コードは原則として LF を使用する。
- OS ごとの自動変換により不要な差分が出ないよう注意する。
- 改行コードのみが変化した差分をそのままコミットしない。

## 3. ファイル名・ディレクトリ名
- ファイル名およびディレクトリ名は、原則として英数字を使用する。
- スペース、日本語、機種依存文字、環境依存文字はできるだけ使用しない。
- 大文字小文字の違いだけによる名前変更は避ける。
- Windows / Mac 間で扱いが不安定になる命名は避ける。

## 4. パスの扱い
- Windows / Mac でパス区切りや大文字小文字の扱いが異なることを意識する。
- パスの直書きは最小限にする。
- 必要に応じて共通処理や定数化を行う。
- 絶対パス依存の実装は避ける。

## 5. エディタ設定差異への注意
- エディタや IDE の設定差異による大量差分をそのままコミットしない。
- 保存時の自動整形や改行コード変換による差分に注意する。
- チームで使用するエディタ設定ファイルがある場合は、それに従う。
