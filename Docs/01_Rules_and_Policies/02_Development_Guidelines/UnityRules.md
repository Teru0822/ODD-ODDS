# Unity 開発で特に守ること (Unity Rules)

Unity（本プロジェクト指定のバージョン）を用いたゲーム開発において、事故や競合を防ぐための注意事項。

## 1. `.meta` ファイル
- `.meta` ファイルは Unity 管理上重要であるため、勝手に削除・再生成しない。
- アセットの移動やリネーム時は `.meta` の整合性に注意する。
- 不要に大量の `.meta` 差分を出さない。

## 2. Scene / Prefab
- Scene と Prefab は競合しやすいため、複数人で同時編集しそうな場合は担当を明確にする。
- 大規模な Scene 変更は、できるだけ小分けにする。
- 関係ないオブジェクトの位置変更や Inspector 差分を含めないよう注意する。
- Prefab の Apply / Revert は意図を確認してから行う。
- Scene、Prefab、ProjectSettings を変更する場合は、関係のない差分を含めないよう特に注意する。

## 3. ProjectSettings / Packages
- `ProjectSettings` や `Packages/manifest.json` の変更は影響が大きいため、必要性を明記する。
- 依存追加・更新は勝手に行わない。
- Unity バージョン差異による自動差分に注意する。

## 4. アセット
- 重いバイナリファイルの追加は最小限にする。
- 不要なアセットを大量にコミットしない。
- 仮素材、テスト素材、出力物は置き場所を分ける。
- 実運用に不要な個人用ファイルはコミットしない。

## 5. 個人環境依存ファイル
- 個人のローカル設定やキャッシュに由来するファイルはコミットしない。
- Unity が自動生成する一時ファイルやビルド生成物は必要がない限り管理対象に含めない。
- `.gitignore` の内容を守る。
