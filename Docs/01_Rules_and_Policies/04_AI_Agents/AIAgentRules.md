# AI エージェントの行動ルール (AI Rules)

AI エージェントが本プロジェクトで開発・改修を行う際に守るべき必須ルールです。

## 行動規範一覧
1. まず関連ファイルを確認し、既存実装を理解する。
2. 指示された範囲に集中して修正する。
3. 無関係な修正や過剰な最適化をしない。
4. 人が読む説明、コメント、タスク記述は日本語で書く。
5. コーディング規約は `Docs/01_Rules_and_Policies/02_Development_Guidelines/CodingRules.md` に従う。
6. 大きな変更を行う場合は、変更方針が分かる形で整理する。
7. 変更後は、内容・理由・対象ファイル・確認事項・懸念点を報告する。
8. 動作未確認なのに「問題なし」と断定しない。
9. シークレット情報、API キー、個人情報を書き込まない。
10. Unity 特有の差分は、意図しない変更が混ざっていないか注意する。
11. 依頼範囲外の Inspector 調整や自動差分を勝手に確定しない。
12. 既存コードが規約から外れていても、依頼範囲外の全面修正は行わない。
13. 文字コードや改行コードの変化だけによる不要差分を発生させない。
14. Windows / Mac の混在環境を前提に、環境依存の記述を避ける。
15. プロンプト作業（タスク）が終了したら毎回、他人に進捗報告ができるよう「どの部分をどう変えたか」「新たに何が出来るようになったか（影響）」を分かりやすくまとめること。
16. AIのアシスタント機能として、ユーザーから各種検証・支援コマンド（`/test-docs`, `/verify-rules`, `/check-hub`, `/check-licenses`, `/create-pr-memo`, `/find-todo`, `/cleanup-temp`, `/summary-report`）が入力された際は、対応するワークフロー（`.agents/workflows/` 配下）を実行し、自発的に検証の実行やプロンプト情報の自動整理を行って報告すること。
