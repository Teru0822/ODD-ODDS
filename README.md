# コンセプト

目標：

- 短期間で作れるようなシンプルなゲーム性
- ワールドは小さく(1部屋)
- ギャンブル要素で中毒性のあるゲームデザイン

**目指す方向性:** 成長要素を追求し、魅力的なゲームループを実現  
  
背景音:
- **BGM:** オルゴール音を基調に、不協和音を含む暗い雰囲気も
- **SE:** 適切な効果音：[Sound Effect Generator](https://soundeffectgenerator.org/ja/sound-effects) を使用

---

**細かい改善要素**
- 持てるアイテム個数を制約する。
- 特典要素としてアチーブメントを実装。

---

**進行:**

### Prototype
  - 核「パック購入」「カードを置く」Cycle完成まで **Early Entry Stage**



### 1. プロトタイプ（ゲームの核になる部分）

**目標:** 「パックを買って，カードを引き，コマが移動し，所持金が変動する」という基本ループが仮の図形で動く状態．

- **プロジェクト構築:** Unityプロジェクトの立ち上げ，基本フォルダ構成の作成．
- **基本システム（スクリプト）:**
    - お金の管理システム（所持金の増減、ゲームオーバー判定）．
    - カードのデータ構造作成（お金、アイテム、移動の3種）．
    - パック開封・抽選ロジックの実装（必ず1枚は移動カードを含める処理）．
- **移動システム:**
    - グリッド状の盤面（仮のCube等）の生成．
    - 移動カードの処理ロジック（**下、左、上、右**の指定方向への移動とマス数計算）．
- **基本UI:** 仮のボタン（パック購入）と，仮のテキスト（所持金、引いたカードの結果）の表示．

### 2. アルファ（全要素が仮素材で実装されている状態）

**目標:** アイテムの使用，強化要素，カメラワークなど，ゲームの全ルールが（見た目はともかく）通して遊べる状態．

- **アイテム＆インベントリシステム:**
    - 机の上のカード置き場（上限10枚）の実装．
    - 上限時のカード交換（破棄）ロジックの実装．
    - 各アイテムカードの効果実装．
- **ローグライク（成長）要素:**
    - きんちゃく袋をクリックした際の強化画面UIの表示．
    - お金を消費してのステータスアップ（パックの質向上など）処理．
- **カメラ＆視点制御:**
    - プレイヤーのメイン視点（机全体）と盤面ズームアップの切り替え処理．
    - パック開封後の強制ズーム＆移動演出ロジック．
    - 人形の一人称視点をRender Texture等でモニター（仮オブジェクト）に映す処理．
- **ゲーム進行管理:** スタート，ゲームオーバー，クリア（ゴールの判定）の遷移を繋ぎ合わせる．

### 3. ベータ（全要素が実装され、グラフィック・サウンドが乗っている状態）

**目標:** AI生成画像やアセットを組み込み，ライティングや演出を加えて「製品の見た目」にする状態．

- **アセット＆グラフィック組み込み:**
    - 3Dモデルの配置（机，悪魔のモチーフ，モニター，きんちゃく袋，ミニチュア人形，迷路キット）．
    - AIやアセットを利用したカードのイラスト・UIデザインの適用．
- **ライティングとフォグの調整:**
    - 通常時のPopな空間のライティング設定．
    - ゲームオーバー時の「真っ暗になり絶望感を出す」照明切り替えとフォグの演出．
- **アニメーション・VFX:**
    - パックを開封する時のエフェクト(癖のある動作にする)．
    - 人形が移動する際のスムーズなアニメーション(癖のある動作にする)．
    - カメラのズームイン・アウト時のスムーズなトランジション（Cinemachine等の活用）．
- **サウンド（BGM/SE）:**
    - 環境音，システム音（パック開封音，カード使用音，お金の音），BGMの実装．

### 4. マスターアップ（デバッグとバランス調整が完了した状態）

**目標:** プレイヤーが理不尽に感じないようバランスを整え、バグを潰してリリースできる状態．

- **ゲームバランスの調整:**
    - パックの価格、排出確率、お金カードの金額の調整．
    - 強化（ローグライク要素）のコストと恩恵のバランス調整．
    - 迷路の長さや難易度の調整．
- **デバッグ・最適化:**
    - カード上限時の挙動や、特定操作時の進行不能バグの洗い出しと修正．
    - 不要なアセットの削減、処理負荷の軽減（特に一人称カメラのレンダリング負荷など）．
- **ビルドと最終確認:** 実行ファイルを出力し、通しプレイでの最終チェック．



# 企画書

**プロジェクト名（仮）：**
**想定プラットフォーム：** PC

**1. コンセプト・概要**
• **ストーリー：** 多額の借金を背負った主人公が，悪魔の持ちかけた「クリアすれば億万長者」のゲームに魂を賭けて挑む．
• **コア体験：** 「机の上」という1部屋の閉鎖空間で完結する，ローグライク要素を持たせた中毒性の高いすごろくギャンブル．
• **アートディレクション：** 通常時はPopでトイライクな雰囲気．ゲームオーバー時は一転してライティングを落とし，深いフォグで絶望感を演出する．

**2. コア・ゲームループ**
プレイヤーは以下のサイクルを繰り返し，ゴールを目指す．
1. **盤面確認・強化：** 机上の状態を確認し，必要に応じて「きんちゃく袋」から能力強化（ローグライク要素）を行う．
2. **パック購入：** 所持金を消費してカードパック（5枚入り）を購入する．
3. **アイテム使用：** パックを開封する前に，手札にあるアイテムカードを任意で使用する．
4. **パック開封・即時発動：**お金カードが出た場合，即座に所持金に加算される．
    ◦ 移動カードが出た場合，強制的に盤面にカメラがズームし，指定された方向へ人形が移動する．
    ◦ アイテムカードが出た場合，手札に加わる（上限10枚，超過時は破棄または入れ替え）．
5. **判定：** ゴール到達（クリア），または所持金ゼロ（ゲームオーバーほかにもあり？）の判定を行う．

**3. システム詳細
3.1 カード仕様**
1パック5枚入り．必ず1枚以上の「移動カード」が含まれる仕様とする．**カード種類発動タイミング効果・詳細お金カード**引いた瞬間（即時）書かれている数字分，所持金が増加する．**移動カード**引いた瞬間（即時）記号と数字に従い人形が移動する．(この際，盤面を強制的にズームさせ，移動させる)内部の方向データ定義は「下，左，上，右」の順を基準に設計する．**アイテムカード**任意（開封前）手札に入る（上限：机に置ける10枚まで）．状況を有利にする様々な効果（例：次の移動量＋1，引き直し等）．
**3.2 視点・カメラワーク（Camera Control）**
プレイヤーは自由に首を振って机の上を見渡せる．各オブジェクトへのフォーカスで視点が切り替わる．
• **メインカメラ（全体視点）：** 机全体（パック，手札，きんちゃく袋，迷路キット）を見下ろす基本視点．
• **ズームカメラ（盤面視点）：** パックから移動カードが引かれた際，強制的に迷路キットにズームインする．
• **モニター視点（Render Texture）：** 迷路内の人形の頭上に配置したカメラの映像を，机の上のモニターオブジェクトに出力する．
**3.3 ローグライク（成長）要素**
机上の「きんちゃく袋（お金）」オブジェクトをクリックすることで専用UIを開き，所持金を消費して恒久的な強化を行う．
• **強化項目の例：**
    ◦ 高レアリティカード（移動量が多い，お金が多い）の排出確率アップ．
    ◦ パック購入費用の割引．
    ◦ 手札の上限拡張（10枚 → 12枚など）．
**3.4 勝利・敗北条件**
• **クリア（勝利）：** 迷路の左端からスタートした人形が，右端のゴール地点へ到達する．
• **ゲームオーバー（敗北）：** 所持金が「0以下」になり，かつパックを買う手段がなくなること．

**4. 必要なオブジェクト・アセット一覧**
• **3Dモデル：** 机，悪魔(見えないようにする？)，モニター，きんちゃく袋，迷路の盤面，ミニチュア人形．
• **2D/UI（AI・アセット活用）：** カードの絵柄（お金，移動方向アイコン，アイテム各種），パックのパッケージ，所持金表示テキスト，強化画面UI．
• **エフェクト・ライティング：** パック開封パーティクル，Popなメイン照明，ゲームオーバー用の暗転・スポットライト・環境フォグ．
